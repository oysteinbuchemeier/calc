📄
Context size
The single biggest lever. Doubling the amount of text you send per request doubles your input cost. This is why RAG architectures and caching strategies matter so much.
🤖
Agent depth & orchestration
Each step in a multi-step agent is a separate API call. A 6-step workflow is 6× the token cost of a single call — plus context accumulates across steps.
🔄
Retrieval strategy
Full-document retrieval vs. chunk-based RAG can change your input cost by 10–50×. How you fetch context matters as much as what model you use.
📝
Output length
Output tokens cost 4–5× more per token than input tokens on most models. Tasks that generate long documents hit this hard — drafting and documentation especially.
⚡
Workflow volume & frequency
Cost scales linearly with volume. 500 runs/day is 10× the cost of 50/day — but also 10× the productivity gain. The ratio matters more than the absolute number.
🧠
Model selection per task
Most enterprise deployments use 2–3 models. Haiku for classification and routing, Sonnet for reasoning and drafting, Opus for high-stakes decisions. Routing correctly reduces cost 40–70%.
